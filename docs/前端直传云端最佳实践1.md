**大文件的切片、并发控制、合并甚至是断点续传，阿里云和 Amazon S3 的前端 SDK 都已经帮你封装好了。** 前端代码不需要手动去切割 `Blob` 或 `File` 对象，也不需要自己写 `Promise.all` 去并发发请求。

为了让你在接下来的开发和后端联调中游刃有余，我为你梳理了这套架构的核心知识点、前后端的分工，以及落地的最佳实践。

---

### 一、 核心概念补齐

在对接 SDK 之前，我们需要弄清几个云存储的基础概念：

1. **Bucket（存储空间/桶）：** 云端的一个逻辑容器，你可以把它理解为云端的一个“顶级文件夹”或者“硬盘分区”。你的文件最终都会存放在某个指定的 Bucket 中。后端在配置云服务时会创建好这个 Bucket，并把它的名称（Bucket Name）和所在地域（Region）告诉你。
2. **Object/Key（对象/文件路径）：** 在 Bucket 内部，文件被称为 Object。文件的完整路径（包含文件名）被称为 Key。例如 `uploads/2026/avatar.png`，这就是一个 Key。
3. **直传（Direct Upload）：** 文件数据流直接从用户的浏览器发送到阿里云或 AWS 的服务器，**完全不经过你们公司的后端服务器**。这极大减轻了你们后端服务器的带宽和内存压力，这也是目前企业级应用处理大文件上传的标配方案。
4. **STS（Security Token Service，临时访问凭证）：** 这是前后端交互的**核心**。云存储的读写是需要权限的。为了安全，绝对不能把云账号的长期秘钥（AccessKey/SecretKey）写在前端代码里。后端需要调用云厂商的接口，生成一个有时效性、权限受限的临时凭证（STS Token）发给前端，前端拿着这个凭证去初始化 SDK。

---

### 二、 前端与后端的分工与交互流程

既然文件不经过后端，那前端和后端在这个场景下需要做什么？

#### 1. 后端需要做的事情

* **提供获取临时凭证（STS）的接口：** 每次前端准备上传文件前，后端需要生成并返回 STS Token、临时 AccessKeyId、临时 AccessKeySecret。
* **提供业务数据的保存接口：** 前端把文件传到云端后，云端只会保存文件。后端需要提供一个接口，让前端把云端返回的**文件 URL 或 File Key** 提交给后端，后端将其与具体的业务数据（比如某个用户的头像字段、某个业务单据的附件字段）绑定并存入数据库。
* **（可选）配置跨域（CORS）：** 云存储的 Bucket 必须配置允许来自你们前端域名的跨域请求，否则浏览器的上传请求会被拦截。这通常是后端或运维在云控制台完成的。

#### 2. 前端需要做的事情

* **请求凭证：** 在用户选择文件准备上传时，先调用后端接口获取 STS 临时凭证。
* **实例化 SDK：** 拿着获取到的 STS 凭证、Bucket 名称、Region 等信息，实例化阿里云（`ali-oss`）或 Amazon S3 的客户端对象。
* **调用上传 API：** 调用 SDK 提供的上传方法（对于大文件，通常调用 `multipartUpload` 相关的 API）。
* **状态反馈与业务提交：** 监听 SDK 返回的进度（Progress）更新 UI。上传成功后，拿到云端返回的 URL/Key，调用后端的业务接口进行保存。

#### 3. 完整交互时序图

1. **前端** -> 用户选择了一个 500MB 的视频文件。
2. **前端** -> 请求后端接口 `/api/upload/get-token`。
3. **后端** -> 请求阿里云/AWS，获取临时 STS 凭证，返回给前端。
4. **前端** -> 使用 STS 凭证初始化 `OSSClient` 或 `S3Client`。
5. **前端** -> 调用 SDK 的分片上传 API `client.multipartUpload(file)`。
6. **前端** -> (SDK 自动处理切片、并发发送到云端、最后通知云端合并)。
7. **云端** -> 返回最终的文件访问 URL 给前端。
8. **前端** -> 将 URL 发送给后端业务接口 `/api/business/save-attachment`。
9. **后端** -> 将 URL 存入数据库，流程结束。

### 三、 最佳实践：基于环境的适配器模式封装

由于你们需要在开发环境使用阿里云，其他环境使用 S3，对于这种场景，最佳实践是**在前端采用“适配器模式（Adapter Pattern）”来抹平 SDK 的差异**。

作为前端，你应该向上层业务组件暴露一个统一的 Hook（例如 `useCloudUpload`），而在 Hook 内部根据环境变量动态选择具体的上传策略。

#### 伪代码架构参考

**1. 定义统一的上传适配器接口 (TypeScript 示例)**

```typescript
interface UploadAdapter {
  // 统一的上传方法签名
  upload: (
    file: File, 
    onProgress?: (percent: number) => void
  ) => Promise<{ url: string; key: string }>;
}

```

**2. 实现阿里云 OSS 适配器**

```typescript
import OSS from 'ali-oss';

export class AliOSSAdapter implements UploadAdapter {
  private credentials: any; // 从后端获取的 STS

  constructor(credentials: any) {
    this.credentials = credentials;
  }

  async upload(file: File, onProgress) {
    const client = new OSS({
      region: this.credentials.region,
      accessKeyId: this.credentials.tempAccessKeyId,
      accessKeySecret: this.credentials.tempAccessKeySecret,
      stsToken: this.credentials.securityToken,
      bucket: this.credentials.bucket,
    });

    // OSS 的分片上传 API，内部已包含切片、并发和合并逻辑
    const result = await client.multipartUpload(file.name, file, {
      progress: (p, cpt, res) => {
        if (onProgress) onProgress(Math.floor(p * 100));
      }
    });

    // 可以在这里处理返回的 URL 格式
    return { url: result.res.requestUrls[0], key: result.name };
  }
}

```

**3. 实现 Amazon S3 适配器**
*注意：AWS SDK 的 API 与 OSS 有所不同，可以使用 `@aws-sdk/client-s3` 和 `@aws-sdk/lib-storage` (专门处理大文件分片)。*

```typescript
import { S3Client } from "@aws-sdk/client-s3";
import { Upload } from "@aws-sdk/lib-storage";

export class AWSS3Adapter implements UploadAdapter {
  // 初始化逻辑类似，使用后端的 STS 初始化 S3Client
  // ...
  
  async upload(file: File, onProgress) {
      const parallelUploads3 = new Upload({
        client: this.s3Client,
        params: { Bucket: this.bucket, Key: file.name, Body: file },
        queueSize: 4, // 并发数
        partSize: 1024 * 1024 * 5, // 5MB 一个分片
      });

      parallelUploads3.on("httpUploadProgress", (progress) => {
        if (onProgress && progress.total) {
            onProgress(Math.floor((progress.loaded / progress.total) * 100));
        }
      });

      await parallelUploads3.done();
      return { url: `https://${this.bucket}.s3.amazonaws.com/${file.name}`, key: file.name };
  }
}

```

**4. 封装统一的 Hook**

```typescript
export const useCloudUpload = () => {
  const [isUploading, setIsUploading] = useState(false);
  const [progress, setProgress] = useState(0);

  const uploadFile = async (file: File) => {
    setIsUploading(true);
    try {
      // 1. 向后端请求对应的 STS 凭证
      const credentials = await fetchStsTokenFromBackend();
      
      // 2. 根据环境变量选择适配器
      let adapter: UploadAdapter;
      if (import.meta.env.VITE_APP_ENV === 'development') {
        adapter = new AliOSSAdapter(credentials);
      } else {
        adapter = new AWSS3Adapter(credentials);
      }

      // 3. 执行上传
      const { url } = await adapter.upload(file, (p) => setProgress(p));
      
      return url;
    } finally {
      setIsUploading(false);
    }
  };

  return { uploadFile, isUploading, progress };
};

```
